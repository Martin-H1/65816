This is a summary of our previous session writing an ITC threaded ANS Forth
(ANSI 1994) for the 65816 CPU. Since the 65816 is natively 16-bit, a 16-bit
cell size is used for speed and simplicity.

The first target board is a single board computer with the w65c265 with
32KB RAM and 32KB ROM. The Forth kernel will be located in the ROM, and the
user will interact using the chip's UART serial interface.

The memory map of the system is:
$0000 - $00FF   Direct Page (Forth VM registers, scratch)
$0100 - $01FF   Hardware stack (Return Stack)
$0200 - $03FF   Parameter Stack (grows downward)
$0400 - $04FF   Scratch PAD area
$0500 - $05FF   User Area
                +$00  BASE         numeric base (default 10)
                +$02  STATE        0=interpret, 1=compile
                +$04  DP           dictionary pointer
                +$06  LATEST       latest definition link
                +$08  TIB          terminal input buffer address
                +$0A  >IN          parse offset
                +$0C  SOURCE-LEN   current source length
                +$0E  HANDLER      exception handler
                +$10  SAVEDP       saved DP for definition rollback
                +$12  SAVELATEST   saved LATEST for definition rollback
$0600 - $06FF   Terminal Input Buffer (TIB)
$0700 - $7EFF   RAM Dictionary (user definitions grow upward)
$8000 - $FFFF   ROM (kernel primitives, core word set)

Note: The debug build is loaded at $4000-$7FFF to allow testing in RAM.

The Forth is written using the CA65 assembler with the following register
allocations:
  X  = Parameter Stack Pointer (PSP)
  Y  = Instruction Pointer (IP)
  A  = Working register
  S  = Return Stack Pointer (RSP) - use the hardware stack!
  D  = Direct Page base for Forth internals
  DB = Data Bank currently zero through the interpreter.

During our previous sessions we ran into a few issues with invalid 65816
op codes, CA65 syntax, and 65816 best programming practices. I've retained a
summary of some of those issues:

The valid 65816 branch instructions for future reference:
Condition    Unsigned              Signed
=            BEQ                  BEQ
≠            BNE                  BNE
<            BCC                  BMI (with overflow check)
≥            BCS                  BPL (with overflow check)
>            reverse operands + BCC  reverse operands + BMI
≤            reverse operands + BCS  reverse operands + BPL

The 65816 CMP only supports immediate, direct page, absolute, and indexed
memory operands - never a register directly. For comparing against Y or X
you need to either use CPY/CPX with a memory or immediate value, or transfer
with TYA/TXA first.

On the 65816, arithmetic and logic instructions (ADC, SBC, AND, ORA, EOR, etc.)
only operate on memory operands or immediate values - never directly on X or Y
registers. To use X or Y as a source you always need TXA/TYA first, or store to
a zero page location and use that.

The CA65 assembler doesn't support escaping the " character in string
constants. This caused problems using the HEADER macro for words with " in the
name (e.g. ABORT"). To resolve this we skipped the HEADER macro for these words
and wrote header bytes manually, including the quote character as a numeric
byte value $22.

Word definitions end in NEXT, so a word definition can't JSR directly to another
word definition. That's because the JSR requires an RTS return. When this is
required, this is worked around by using the RTS_CFA_LIST trampoline.

TONUMBER_CODE calling UMSTAR_CODE is an example of this pattern. An alternate
approach is to split the definition into a separate _IMPL subroutine that both
definitions can JSR to. UMSLASHMOD uses this approach.

The 65816 supports stack relative addressing which is used to create stack local
variables within a procedure. This reduces the need for direct page space, and
eases register starvation issues. A restriction is that the only indirect stack
local addressing mode is indexed indirect addressing using the Y register like
this: "lda (OFFSET,S),Y".

In addition, the 65816 supports moving the direct page pointer to the
procedure's stack frame. This is useful for using direct page addressing modes
with stack locals. This technique is used in the ACCEPT_CODE function and in
the preferred coding style sample. My coding prompts will contain detailed
requirements around using this technique.

However, any primitive that uses zero page variables must have D=$0000, and
any caller that has D pointing to a stack frame must save/restore D around
such calls. TONUMBER_CODE is an example of this technique because it calls
UMSTAR which uses page zero scratch variables. Calls via the RTS_CFA_LIST
trampoline must ensure D=$0000. If needed precede by PHD/LDA #0/TCD and
follow by PLD to ensure NEXT has D=$0000.

Here are additional 65816 gotchas:
STZ only supports direct page and absolute addressing — no indirect modes.
Use LDA #0 / STA (addr) instead.
INC, DEC, LDY, LDX don't support stack-relative addressing — only LDA/STA do.
JSR indirect doesn't exist — use JMP indirect instead (safe since Forth words
end in NEXT anyway).
BGE, BLE, BLT, BGT don't exist — see the branch table above in the summary.
PEA $0000 / PLB / PLB is a cleaner way to set the Data Bank Register than
SEP / LDA #0 / PHA / PLB.

PHY/PLY around loops:

* Any primitive that uses LDY #n or TAY as a loop counter must save and restore
  Y with PHY/PLY since Y holds the Forth instruction pointer (IP). However, when
  Y is saved in the stack frame this can be omitted.
* BEQ/BNE after INX/INX: INX/INX clobbers the zero flag, so you can't test the
  just-popped value with BEQ/BNE directly after. Save to a zero page variable,
  stack local, or use CMP #0 after the pops.

Register discipline for assembly primitives:
A  — free to clobber
X  — parameter stack pointer, must be preserved (or explicitly adjusted for push/pop)
Y  — instruction pointer (IP), must be preserved with PHY/PLY if used
S  — return stack pointer, managed by hardware stack operations
D  — must be $0000 on entry and exit via NEXT
DB — must remain zero

Any routine that clobbers Y is responsible for saving and restoring it with
PHY/PLY. This applies even to internal subroutines called via JSR — the callee
preserves Y, not the caller.

* Double number support with true 32-bit >NUMBER and NUMBER?.

New words implemented and tested:

Double number support:

* NUMBER upgraded to true 32-bit double accumulation.
* NUMBER? upgraded to support trailing dot double literals (e.g. 1234.).
* 2LIT / 2LITERAL — runtime and compile-time double literals.
* D+ D- D= D< DU< — double arithmetic and comparison.
* DABS DNEGATE — double sign operations.
* D. D.R — double output.
* D>S S>D M+ — single/double conversion and mixed arithmetic.
* D0= D0< DMAX DMIN D2* D2/ — double predicates and shifts.
* 2VARIABLE 2CONSTANT 2VALUE — double defining words.
* 2ROT — rotate three double numbers.
* M*/ — mixed multiply-divide.

CORE-EXT words implemented:

* .( — print string immediately, works in both interpret and compile mode.
* .R U.R — right-justified signed and unsigned output.
* ERASE — fill memory with zeros.
* WITHIN — range check.
* BUFFER: — create a named buffer.
* [COMPILE] — force compilation of next word regardless of immediate flag.
* HOLDS — prepend string to pictured numeric output.
* VALUE / TO — mutable named values with safety check (aborts if TO used on
  CONSTANT).
* 2VALUE — double mutable named value.
* C" — counted string literal (compile semantics only; interpret semantics
  undefined by ANS).
* UNUSED — available dictionary space.

Compiler security:

* ?PAIRS ( n1 n2 -- ) — aborts with "mismatched control structure" if values
  differ or stack underflow. Must be F_IMMEDIATE.
* Compiler security numbers defined in constants.inc:
    CS_BEGIN_AGAIN  = 1
    CS_IF_ELSE_THEN = 2
    CS_DO_LOOP      = 3
    CS_WHILE_REPEAT = 4
    CS_CASE_ENDCASE = 5
    CS_OF_ENDOF     = 6
* All control structures (IF/ELSE/THEN, BEGIN/UNTIL/AGAIN/WHILE/REPEAT,
  DO/?DO/LOOP/+LOOP, CASE/OF/ENDOF/ENDCASE) push and verify security numbers.

Dictionary rollback:

* CHECKPOINT ( -- ) saves DP and LATEST to U_SAVEDP and U_SAVELATEST.
* ROLLBACK ( -- ) restores DP and LATEST from checkpoint, clears checkpoint.
* COMMIT ( -- ) clears checkpoint after successful definition.
* do_checkpoint / do_rollback — internal subroutines called via JSR from
* DOCREATE and ABORT respectively. Both preserve Y with PHY/PLY internally.
* DOCREATE calls do_checkpoint automatically via JSR.
* REVEAL is now a DOCOL definition that calls COMMIT after clearing F_HIDDEN.
* ABORT calls do_rollback via JSR before resetting stacks.
* DOVAL / DO2VAL — separate code pointers from DOCON to allow TO safety
  checking. TO checks the code pointer of the target word and aborts if not
  DOVAL or DO2VAL.

New macros:

* IFETCH — LDA 0,Y / INY / INY — fetch inline cell from instruction stream.
* PUSHIP — pushes Y (IP) onto parameter stack.
* IPFETCH — IFETCH followed by PUSH.
* POPIP — pops TOS into Y (IP).
* CELL+ — add one cell size to TOS (replaces LIT 2 PLUS pattern).

New HAL functions:

* hal_nputs ( addr Y=count -- ) — print N characters from address, Y is

Important bugs found and fixed:

* WORD was not updating >IN correctly after TCD relocated the direct page.
  Fix: use (LOC_UP),Y with Y=#U_TOIN instead of STA (U_TOIN).
* #<QUIT_BODY in ABORT_CODE loaded only the low byte of the address.
  Fix: use #QUIT_BODY for the full 16-bit address.
* (DOES>) was destroying A with AND #1 alignment test before storing the CFA
  address. Fix: save tentative CFA to SCRATCH0 first, then test alignment and
  increment SCRATCH0 if odd.
* CREATE must reserve a placeholder cell at CFA+2 for the DOES> code address,
  otherwise , overwrites it with body data. Body data starts at CFA+4.
* ca65 linker bug: hal_mench.o must be listed first in the linker command or
  hal_cputs resolves incorrectly. Document in Makefile.
* RECURSE/LATEST>CFA — refactored using U_CURDEF in DOCREATE.
* DOMARKER_CODE was clobbering Y (the IP) with LDY instructions. Fix: wrap with
  PHY/PLY.
* /MOD was implementing symmetric division instead of floored. Fix: rewritten
  as DOCOL using S>D and FM/MOD.
* DOABORTQUOTE inline string length must be a 16-bit word cell, not a byte.
  hal_nputs uses Y as a word-sized length to handle this correctly.
* S" in interpret mode stores strings in PAD which gets overwritten by
  subsequent S" calls. Tests that need two strings simultaneously must use C"
  or CREATE inside word definitions. C" interpret semantics are undefined by
  ANS; only use C" in word definitions.

Coding style rules established:
Three classes of routines each with their own pattern:
Simple loops — use PHY/PLY with Y as counter only. See SPACES_CODE.
Buffer walks — push addr and count onto hardware stack, use (offset,S),Y with
Y as byte index. See TYPE_CODE.
Complex routines — use full PHD/TCD stack frame. All temporaries in the frame.
No SCRATCH0/SCRATCH1/TMPA/TMPB/W after TCD. See NUMBER_CODE in
sample_coding_style.s.

Additional rules:

* After TCD, all parameter stack accesses via X must use absolute forced
  addressing: LDA a:0,X.
* Zero page values needed throughout a routine must be fetched into locals
  immediately after reserving the frame but before TCD.
* Result must be pushed to parameter stack before frame teardown since TSC
  clobbers A.
* Use $FF ^ F_HIDDEN not ~F_HIDDEN for byte complement — ca65 produces a large
  constant with ~.
* BRANCH and ZBRANCH use absolute addresses not offsets for branch targets.
* Do not use numeric literals for character codes; use named constants such as
  C_RETURN, BKSP, DEL, SPACE, L_FEED defined in constants.inc.
* Use a single @return label with BRA from success and fall-through from
  failure, with stack teardown done once.
* Result push order: DEX/DEX, STA a:0,X, TSC/CLC/ADC/TCS, PLY, PLD, NEXT.
* Flags set by the last operation should be exploited where possible.
* Any routine that uses LDY clobbers Y (the IP) and must save/restore with
  PHY/PLY. This is the callee's responsibility, not the caller's.
* ABORT" strings use a 16-bit word length cell followed by string bytes,
  aligned. to even boundary. Use ?PAIRS with LIT_CFA before the security
  number constant.
