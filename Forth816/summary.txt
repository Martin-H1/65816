This is a summary of our previous session writing an ITC threaded ANS Forth
(ANSI 1994) for the 65816 CPU. Since the 65816 is natively 16-bit, a 16-bit
cell size is used for speed and simplicity.

The first target board is a single board computer has with the w65c265 with
32KB RAM and 32 KB ROM. The Forth kernel will be located in the ROM, and the
user will interact using the chip's UART serial interface.

The memory map of the system is:
$0000 - $00FF   Direct Page (Forth VM registers, scratch)
$0100 - $01FF   Hardware stack (Return Stack)
$0200 - $03FF   Parameter Stack (grows downward)
$0400 - $04FF   User Area
                +$00  BASE         numeric base (default 10)
                +$02  STATE        0=interpret, 1=compile
                +$04  DP           dictionary pointer
                +$06  LATEST       latest definition link
                +$08  TIB          terminal input buffer address
                +$0A  >IN          parse offset
                +$0C  SOURCE-LEN   current source length
                +$0E  HANDLER      exception handler
$0500-$05FF     Terminal Input Buffer (TIB)
$0600-$7EFF     RAM Dictionary (user definitions grow upward)
$8000 - $FFFF   ROM (kernel primitives, core word set)

The Forth is written using the CA65 assembler with the following register
allocations:
  X  = Parameter Stack Pointer (PSP)
  Y  = Instruction Pointer (IP)
  A  = Working register
  S  = Return Stack Pointer (RSP) - use the hardware stack!
  D  = Direct Page base for Forth internals
  DB = Data Bank currently zero through the interpreter.

During our previous sessions we ran into a few issues with invalid 65816
op codes, CA65 syntax, and 65816 best programming practices. I've retained a
summary of some of those issues:

The valid 65816 branch instructions for future reference:
Condition    Unsigned  	     	    Signed
=	     BEQ	     	     BEQ
≠	     BNE	     	     BNE
<	     BCC	     	     BMI (with overflow check)
≥	     BCS	     	     BPL (with overflow check)
>	     reverse operands + BCC  reverse operands + BMI
≤	     reverse operands + BCS  reverse operands + BPL

The 65816 CMP only supports immediate, direct page, absolute, and indexed
memory operands - never a register directly. For comparing against Y or X
you need to either use CPY/CPX with a memory or immediate value, or transfer
with TYA/TXA first.

On the 65816, arithmetic and logic instructions (ADC, SBC, AND, ORA, EOR, etc.)
only operate on memory operands or immediate values - never directly on X or Y
registers. To use X or Y as a source you always need TXA/TYA first, or store to
a zero page location and use that.

The CA65 asssembler doesn't support escaping the " character in string
constants. This caused problems using the HEADER macro for words with " in the
name (e.g. ABORT"). To resolve this we skipped the HEADER macro for these words
and wrote header bytes manually, including the quote character as a numeric
byte value $22.

Word defintions end in NEXT, so a word definition can't JSR directly to another
word definition. That's because the JSR requires and RTS return. When this is
required, this is worked around by using the RTS_CFA_LIST trampoline.
TONUMBER_CODE calling UMSTAR_CODE is example of this pattern.

The 65816 supports stack relative addressing which is used to create stack local
variables within a procedure. This reduces the need for direct page space, and
eases register starvation issues. A restriction is that the only indirect stack
local addressing mode is indexed indirect addressing using the Y register like
this: "lda (OFFSET,S),Y".

In addition, the 65816 supports moving the direct page pointer to the
procedure's stack frame. This is useful for using direct page addressing modes
with stack locals. This technique is used in the ACCEPT_CODE function and in
the preferred coding style sample. My coding prompts will contain detailed
requirements around using this technique.

However, any primitive that uses zero page variables must have D=$0000, and
any caller that has D pointing to a stack frame must save/restore D around
such calls. TONUMBER_CODE is an example of this technique.

Here are additional 65816 gotchas:

STZ only supports direct page and absolute addressing — no indirect modes.
Use LDA #0 / STA (addr) instead.
INC, DEC, LDY, LDX don't support stack-relative addressing — only LDA/STA do.
JSR indirect doesn't exist — use JMP indirect instead (safe since Forth words
end in NEXT anyway).
BGE, BLE, BLT, BGT don't exist — see the branch table above in the summary.
PEA $0000 / PLB / PLB is a cleaner way to set the Data Bank Register than
SEP / LDA #0 / PHA / PLB

PHY/PLY around loops:
* Any primitive that uses LDY #n or TAY as a loop counter must save and restore
Y with PHY/PLY since Y holds the Forth instruction pointer (IP). However, when
Y is saved in the stackframe this can be omitted.
* BEQ/BNE after INX/INX: INX/INX clobbers the zero flag, so you can't test the
just-popped value with BEQ/BNE directly after. Save to a zero page variable,
stack local, or use CMP #0 after the pops.

New words implemented and tested:

[ and ] — switch between interpret and compile mode. Both require PHY/PLY
around LDY since Y holds the IP at all times.
COMPILE, — compiles an xt into the current definition. In ITC Forth identical
to ,.
WORDS — rewritten as a DOCOL definition to test the inner interpreter. Uses
dictionary traversal with FETCH, CFETCH, TYPE, and ZBRANCH.
INTERPRET — rewritten as a DOCOL definition. Handles word lookup via FIND,
number conversion via NUMBER, and compiling via COMPILE, and LIT.
QUIT — implemented as a DOCOL definition. The double return stack reset
(once in ABORT, once in QUIT) is intentional and harmless.
UNDEFINED-WORD — prints "error: Undefined word" and calls ABORT.
TRACEON / TRACEOFF — debug words that set a zero page variable TRACE_EN to
enable/disable execution tracing via a modified NEXT macro.
TRACEOUT — prints current IP in hex using DOTHEX_CODE::print_chex. Reuses
existing hex output code.
(CREATE) — hidden primitive that builds a dictionary header from a counted
string at the address on the stack ( addr -- ). Used by :, CREATE, VARIABLE,
and CONSTANT.
: — rewritten as a DOCOL definition using (CREATE).
; — rewritten as a DOCOL definition using REVEAL and [.
REVEAL — assembly primitive that clears F_HIDDEN on the most recent dictionary
entry. Replaces inline flag manipulation in ;, VARIABLE, and CONSTANT.
CREATE — DOCOL definition. Reserves an extra placeholder cell at CFA+2 for the
DOES> code address so that body data starts at CFA+4.
DOES> — immediate DOCOL word that compiles (DOES>) into the current definition.
(DOES>) — hidden assembly primitive. Patches the CFA of the most recently
CREATEd word to DODOES and stores the current IP (DOES> code address) at CFA+2.
DODOES — updated to push CFA+4 as the body address and set IP from CFA+2.
VARIABLE — DOCOL definition using (CREATE), DOVAR, and REVEAL.
CONSTANT — DOCOL definition using (CREATE), DOCON, and REVEAL.
DECIMAL / HEX — DOCOL definitions that store 10 or 16 into BASE.

Important bugs found and fixed:

WORD was not updating >IN correctly after TCD relocated the direct page.
Fix: use (LOC_UP),Y with Y=#U_TOIN instead of STA (U_TOIN).
#<QUIT_BODY in ABORT_CODE loaded only the low byte of the address.
Fix: use #QUIT_BODY for the full 16-bit address. Search for other #< usages
throughout the codebase.
(DOES>) was destroying A with AND #1 alignment test before storing the CFA
address. Fix: save tentative CFA to SCRATCH0 first, then test alignment and
increment SCRATCH0 if odd.
CREATE must reserve a placeholder cell at CFA+2 for the DOES> code address,
otherwise , overwrites it with body data. Body data starts at CFA+4.
ca65 linker bug: hal_mench.o must be listed first in the linker command or
hal_cputs resolves incorrectly. Document in Makefile.

Coding style rules established:
Three classes of routines each with their own pattern:

Simple loops — use PHY/PLY with Y as counter only. See SPACES_CODE.
Buffer walks — push addr and count onto hardware stack, use (offset,S),Y with
Y as byte index. See TYPE_CODE.
Complex routines — use full PHD/TCD stack frame. All temporaries in the frame.
No SCRATCH0/SCRATCH1/TMPA/TMPB/W after TCD. See NUMBER_CODE in
sample_coding_style.s.

Additional rules:

After TCD, all parameter stack accesses via X must use absolute forced
addressing: LDA a:0,X.
Zero page values needed throughout a routine must be fetched into locals
immediately after reserving the frame but before TCD.
Result must be pushed to parameter stack before frame teardown since TSC
clobbers A.
Use $FF ^ F_HIDDEN not ~F_HIDDEN for byte complement — ca65 produces a large
constant with ~.
The FIG Forth "smudge bit" is equivalent to F_HIDDEN. Your REVEAL approach is
cleaner than FIG's toggle-based SMUDGE.
BRANCH and ZBRANCH use absolute addresses not offsets for branch targets.


Debugging tools added:

TRACEOUT in a .ifdef DEBUG block calls DOTHEX_CODE::print_chex to print the
current IP.
trace_decode.py — Python 3 script that reads the CA65 map file and translates
hex IP values in trace output to symbol names with offsets. Parses the "Exports
list by name" section. Subtracts 2 from addresses to compensate for NEXT
pre-increment. Known INTERPRET_BODY offsets are mapped to named labels.
